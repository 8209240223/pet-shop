# 实验二截图清单

## 1. 项目运行首页截图

打开地址：http://192.168.132.128/

截图内容必须包含：
- 浏览器地址栏里的 `192.168.132.128`
- 页面标题“能下单、能管理、能看旧技术和新技术的宠物商店”
- 商品、客户、订单实时统计面板
- 至少一张商品卡片

## 2. 商品加入购物车截图

操作方式：
1. 打开首页。
2. 点击第一张商品卡片的“加入购物车”。
3. 滚动到“购物车”区域。

截图内容必须包含：
- 购物车中有商品名称和数量
- 合计金额不是 `￥0.00`

## 3. Spring Boot 后端配置截图

打开文件：`src/main/resources/application.yml`

截图内容必须包含：
- `server.port: 8080`
- H2 数据库连接配置
- `spring.sql.init.encoding: UTF-8`
- JSP 视图前缀和后缀配置

## 4. Maven WAR 打包配置截图

打开文件：`pom.xml`

截图内容必须包含：
- `<packaging>war</packaging>`
- `spring-boot-starter-web`
- `tomcat-embed-jasper`
- `spring-boot-maven-plugin`

## 5. Nginx 反向代理配置截图

打开文件：`deploy/nginx/petshop.conf`

截图内容必须包含：
- `listen 80`
- `root /usr/share/nginx/petshop`
- `/api/` 反向代理到 `127.0.0.1:8080`
- `access_log` 和 `error_log`

## 6. Docker Compose 配置截图

打开文件：`deploy/docker/docker-compose.yml`

截图内容必须包含：
- `petshop-backend`
- `petshop-frontend`
- 后端端口 `8080:8080`
- 前端端口 `80:80`
- 日志或数据 volume

## 7. Dockerfile 后端截图

打开文件：`deploy/docker/Dockerfile.backend`

截图内容必须包含：
- `tomcat:9.0-jdk17`
- `ROOT.war`
- `EXPOSE 8080`

## 8. Dockerfile 前端截图

打开文件：`deploy/docker/Dockerfile.frontend`

截图内容必须包含：
- `nginx`
- 拷贝 `petshop.conf`
- 拷贝静态资源到 `/usr/share/nginx/html`
- `EXPOSE 80`

## 9. 自动化部署脚本截图

打开文件：`deploy/linux/deploy-petshop.sh`

截图内容必须包含：
- `mvn -DskipTests package`
- 复制 war 到 Tomcat 的命令
- 复制静态文件到 Nginx 的命令
- `nginx -t`

## 10. 日志备份计划任务截图

打开文件：`deploy/linux/backup-logs.sh` 和 `deploy/linux/crontab-example.txt`

截图内容必须包含：
- 备份 `/var/log/nginx`
- 备份 `/opt/tomcat9/logs`
- `find ... -mtime +7 -delete`
- crontab 里的 `30 23 * * *`

在项目目录运行：

```powershell

$env:Path='D:\JDK21\bin;' + $env:Path
& 'C:\Users\王仪杰\Desktop\其他\IntelliJ IDEA 2024.3.1.1\plugins\maven\lib\maven3\bin\mvn.cmd' -DskipTests package
```

截图内容必须包含：
- `BUILD SUCCESS`
- 生成 `pet-shop-learning-1.0.0.war`

## 12. Docker 环境截图

如果电脑安装了 Docker Desktop，截图：

```powershell
docker --version
docker compose version
docker compose -f deploy/docker/docker-compose.yml config
```

截图内容必须包含：
- Docker 版本
- Docker Compose 版本
- Compose 配置检查输出

如果没有安装 Docker Desktop，就不要截“docker 不存在”的错误图；报告里写“本机未安装 Docker Desktop，已完成 Dockerfile 和 Compose 配置设计，容器运行部分在 Linux/Docker 环境中执行”。
