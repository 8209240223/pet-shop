## 5. Nginx 反向代理

服务器配置文件：

```text
/etc/nginx/conf.d/petshop.conf
```

关键配置：

```nginx
server {
    listen 80;
    server_name 192.168.132.128;

    root /usr/share/nginx/petshop;
    index index.html;

    access_log /var/log/nginx/petshop-access.log;
    error_log /var/log/nginx/petshop-error.log;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /legacy/ {
        proxy_pass http://127.0.0.1:8080/legacy/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /h2-console/ {
        proxy_pass http://127.0.0.1:8080/h2-console/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 6. 自动化部署脚本

服务器脚本：

```text
/opt/pet-shop-learning/deploy/linux/deploy-petshop.sh
```

功能：

- 使用 Maven 构建 WAR。
- 将 WAR 复制为 `/opt/tomcat9/webapps/ROOT.war`。
- 重启 `tomcat9` 服务。
- 将静态资源复制到 `/usr/share/nginx/petshop`。
- 检查并重载 Nginx。

## 8. 报告截图建议

1. 在虚拟机终端截 `hostname -I`，证明服务器 IP 是 `192.168.132.128`。
2. 在虚拟机终端截 `java -version && node -v && npm -v && git --version && mvn -v`，证明环境安装完成。
3. 在虚拟机终端截 `/opt/tomcat9/bin/version.sh`，证明实际运行的是 Apache Tomcat 9。
4. 在虚拟机终端截 `systemctl status tomcat9`，证明后端由 Tomcat 9 运行。
5. 在虚拟机终端截 `systemctl status nginx`，证明 Nginx 正在运行。
6. 在虚拟机终端截 `ls -lh /opt/tomcat9/webapps/ROOT.war`，证明 WAR 已部署到 Tomcat。
7. 在虚拟机终端截 `ss -lntp | grep -E ':80|:8080'`，证明 80 和 8080 端口监听。
8. 在虚拟机终端截 `cat /etc/nginx/conf.d/petshop.conf`，证明 Nginx 反向代理到 `127.0.0.1:8080`。
9. 在虚拟机终端截 `curl http://127.0.0.1/api/legacy/stats`，证明 Nginx 代理后的接口可访问。
10. 在 Windows 浏览器截 `http://192.168.132.128/` 页面，地址栏必须是虚拟机 IP，不能是 `localhost`。
